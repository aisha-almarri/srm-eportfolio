Design Proposal

Applying Secure-by-Design Principles

Author: Aisha Almarri | Date: November 2024

Based on readings: Ross (NIST), Salter et al. (1998), and the GitHub Threat Model Cookbook.

This document outlines how we can integrate security from the outset, rather than bolting it on later, creating a foundation for resilient system architecture.

Proposed Security Principles

Principle of Least Privilege

All application components and users will operate with the minimum permissions necessary.

Mitigates: "Elevation of Privilege" threat from Unit 3 threat modeling

Defence in Depth

We will implement multiple layers of security (e.g., WAF, input validation, database encryption) so that a failure in one layer does not lead to a total compromise.

This approach ensures that even if one security control fails, additional layers provide backup protection.

Fail Securely

Our application will default to a "deny" state if an error occurs, preventing unintended access during system failures.

This principle ensures that security is maintained even in edge cases and failure scenarios.

Separation of Duties

Critical actions (e.g., deploying to production) will require approval from two team members.

This prevents single points of failure in critical security processes.

Justification & Integration

Multidisciplinary Approach

As argued by Salter et al., security must be a multidisciplinary effort. By baking these principles into our design from the start (as suggested by the SSE guidelines), we reduce the long-term cost and complexity of managing risk.

Alignment: These principles directly support the ERM concepts from Unit 2 by embedding risk management into the system architecture itself.

Cost-Benefit Analysis

Implementing security early in the design phase is significantly more cost-effective than retrofitting security controls later in the development lifecycle.

  • Early implementation reduces remediation costs by up to 80%
  • Prevents security-related delays in deployment
  • Reduces technical debt associated with security patches
