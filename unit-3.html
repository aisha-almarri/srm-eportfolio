Threat Model Report

Threat Model for Web Application

Author: Aisha Almarri | Date: November 2024

Methodology Used: STRIDE (as per the Threat Modeling Manifesto, 2020, and Shevchenko et al.)

Tool: Microsoft Threat Modeling Tool

STRIDE Analysis Process

Spoofing

Risk of an attacker spoofing a user's identity through compromised credentials or session hijacking.

Tampering

Risk of tampering with data in the database through SQL injection or unauthorized data modifications.

Repudiation

Risk that a user denies performing an action due to inadequate logging and audit trails.

Information Disclosure

Risk of exposing sensitive customer data through insecure APIs or improper access controls.

Denial of Service

Risk of the application being overwhelmed through DDoS attacks or resource exhaustion.

Elevation of Privilege

Risk of a user gaining admin rights through privilege escalation vulnerabilities.

Key Finding

Critical Threat Identified

The most critical threat identified was Information Disclosure via the web server-to-database communication channel. This directly links back to the risk we identified in Unit 2 regarding customer data protection.

This finding emphasized the need for encrypted database connections, proper access controls, and regular security assessments of database communication channels.

Methodology Insights

Data Flow Diagrams (DFD)

Creating DFDs helped visualize the application architecture and identify potential attack vectors at each data flow point.

Systematic Approach

The STRIDE methodology provided a structured framework for analyzing threats across different system components systematically.
